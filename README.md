This repository was created for MHC++'s Spring 2026 Git Gud workshop. This workshop aims to teach the fundamental concepts and workflows behind Git and GitHub collaboration, while allowing you to practice them through guided exercises.

By the end of this workshop, you will know how to:

- Describe the differences between **Git** and **GitHub**
- **Fork** and **clone** repositories
- Work with **collaborators**
- Use core Git commands
- Work with **branches**
- Use a common **Git + GitHub workflow** (push/pull with a `main` + feature branch pattern)
- Understand different **merge types** and resolve **merge conflicts**
- Open and submit a **Pull Request (PR)**

## Core Git Commands

| Command | What it does |
|-------|------|
| `git clone` | Creates a local copy of an existing Git repository |
| `git status` | Shows the current state of your repository |
| `git add <file>` | Adds a file to the staging area |
| `git add .` | Stages all changes |
| `git commit -m "message"` | Records a snapshot of staged changes |
| `git branch` | Lists branches |
| `git branch <name>` | Creates a new branch |
| `git switch <branch>` | Switches to another branch |
| `git switch -c <branch>` | Creates and switches to a new branch |
| `git merge <branch>` | Merges another branch into the current one |
| `git push` | Uploads commits to the remote repository |
| `git pull` | Downloads changes from the remote repository |

*Note: this is not an exhaustive list, but contains the commands we cover in the workshop.*

## Typical GitHub Workflow

A common collaboration pattern for team projects or open source contributions:

1. **Fork** the repository
2. **Clone** it to your machine
3. Create a **feature branch**
4. Make changes and **commit**
5. **Push** your branch to GitHub
6. Open a **Pull Request**
7. Review, merge, and repeat
